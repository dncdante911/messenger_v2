## 🔴 Критические проблемы (Node.js)

### 1. Контроллеры без error handling (крашат worker)

```
JoinController.js          — user_status.status без null-check → crash
IsChatOnController.js      — last_message.seen без null-check → crash
TypingController.js        — нет обработки ошибок DB
MuteController.js          — нет обработки
DeclineCallController.js   — нет обработки
RegisterReactionController — нет обработки
+ ещё ~20 контроллеров
```

**Результат:** одна ошибка MySQL → crash worker → потеря 1/18 пользователей на несколько секунд.

### 2. Блокирующие записи файлов (5 штук)

```javascript
// media-upload.js:127 — ПРИ КАЖДОЙ ЗАГРУЗКЕ МЕДИА:
fs.writeFileSync(absPath, uploadedFile.buffer);
// Видео 1 GB → event loop заблокирован на ~3-5 секунд
// ВСЕ остальные пользователи ждут
```

**Решение:** `await fs.promises.writeFile(absPath, buffer)`

### 3. Нет валидации типов файлов

```javascript
// Принимает что угодно — .exe, .php, .sh
let mediaType = (req.body.type || 'file').toLowerCase();
const filename = generateFilename(uploadedFile.originalname);
fs.writeFileSync(absPath, uploadedFile.buffer);
// Никакой проверки magic bytes / MIME
```

### 4. Rate limiter в памяти (не для cluster)

```
18 workers × отдельный rate limiter = фактический лимит 300 × 18 = 5400 req/min
```

Нужен Redis-backed limiter для корректной работы в PM2 cluster.

```
Железо:  E5-2680v4 ×2 (56 потоков) + 128 GB RAM + NVMe 512 GB + 12 TB HDD + 200 Mbit
PM2:     18 workers (используется 32% ядер)
```

| Ресурс | Доступно | Используется | Оценка |
|--------|----------|:------------:|:------:|
| CPU (56 потоков) | 56 | 18 workers | ⚠️ Можно 36–40 workers |
| RAM | 128 GB | ~2–3 GB (Node) + MySQL + Redis | ✅ Огромный запас |
| NVMe (DB) | 512 GB | ~10–50 GB | ✅ Надолго |
| HDD (медиа) | 12 TB | ? | ✅ Отлично |

## 🚀 Можно ли релизить?

### Ответ: **Почти. 1–2 недели на критические фиксы.**

### Блокеры релиза (по приоритету)

| # | Что | Время | Почему критично |
|:-:|-----|:-----:|-----------------|
| 1 | try/catch в 25+ контроллерах | 1 день | Один кривой запрос → crash worker |
| 2 | `writeFileSync` → `writeFile` (5 штук) | 2 часа | 1 GB видео = 5 сек freeze для ВСЕХ |
| 3 | Magic bytes валидация | 4 часа | Загрузка .exe/.php через API |
| 4 | `isPremium = false` → реальный статус | 2 часа | PRO-юзеры не получают PRO-функции |
| 5 | Followers/Following onClick | 1 час | Кнопки ведут в никуда |
| 6 | Профиль по клику на аватар | 1 час | Базовый UX |
| 7 | Жалобы на спам | 1–2 дня | Google Play требует |
| 8 | Rate limiter → Redis-backed | 4 часа | Cluster mode делает in-memory бесполезным |

> После этого можно релизить для **early adopters** (100–1000 юзеров).

### Для массового релиза (10k+ юзеров) дополнительно:

- Offline-кеш (Room DB) — 1–2 недели
- Исчезающие сообщения — 3–4 дня
- FCM push для звонков — 2–3 дня
- Замена 458 `console.log` на winston/pino — 1 день
- Web-версия — 2–3 месяца
