### 3.1 Ваше железо — оценка потенциала

**Конфигурация:**
- 2× Intel Xeon E5-2680v4 (14 ядер × 2 процессора = 28 физ. / 56 логич.)
- 64 GB DDR4 ECC
- 10 TB HDD (хранение медиа)
- 512 GB NVMe SSD (база данных, Redis, кэш)
- 2× 100 Мбит/с = 200 Мбит/с

**Расчёт реальной ёмкости:**

| Ресурс | Ваши возможности | Что нужно |
|--------|-----------------|-----------|
| Одновременные WebSocket соединения | ~15 000–25 000 (Node.js на 4-8 воркерах) | Зависит от аудитории |
| Сообщений/сек (MySQL + Redis) | ~5 000–8 000 msg/s (SSD) | Для 10K DAU — ~50 msg/s в пике |
| Хранение медиа | 10 TB HDD | Достаточно для старта (10M файлов по ~1MB) |
| Полоса пропускания | 200 Мбит/с ≈ 25 MB/s | Хватает до ~2 000 одноврем. звонков |
| ОЗУ под Node.js + MySQL + Redis | 64 GB | Достаточно для 50K DAU |

**Узкие места:**

| Проблема | Риск | Решение |
|----------|------|---------|
| 1 сервер = Single Point of Failure | 🔴 Критично | Хотя бы 1 резервный узел |
| HAProxy — 1 экземпляр | 🔴 Критично | HA-пара HAProxy + Keepalived |
| HDD для медиа (медленные I/O) | 🟡 Средний | Уже правильно: NVMe под БД, HDD под медиа |
| 200 Мбит/с для видеозвонков | 🟡 Средний | 1 HD-видеозвонок ≈ 2–4 Мбит/с → ~50 пар одноврем. |
| MySQL без репликации | 🔴 Критично | Master-Slave репликация |
| Redis без persistence | 🟡 Средний | Включить AOF или RDB снапшоты |

## ЧАСТЬ 4 — СИЛЬНЫЕ СТОРОНЫ WorldMates

### Что у вас есть и чего нет у конкурентов:

#### 1. Гибридное шифрование (УНИКАЛЬНО)
```
Telegram: Только секретные чаты E2EE, облачные — серверное шифрование
Viber: E2EE по умолчанию, но только одна схема

WorldMates: Одновременно три уровня:
  - cipher_version=1: AES-128-ECB (совместимость с веб)
  - cipher_version=2: AES-256-GCM (основной)
  - cipher_version=3: Signal Double Ratchet (максимальная защита)
  → ЧЕСТНАЯ FORWARD SECRECY, которой у Telegram в облачных чатах нет!
```

#### 2. Платформа ботов с FSM (на уровне Telegram)
```
Telegram: Лучшая в индустрии Bot API
Viber: Нет нормального бот-API
WorldMates: Bot API с FSM, inline-клавиатурами, webhook, опросами, RSS
  → Реально конкурентоспособно с Telegram Bot API!
```

#### 3. Интеграция с социальной сетью
```
Telegram/Viber: Чистые мессенджеры без социальной составляющей
WorldMates: Полная интеграция с worldmates.club
  → Уникальный "мессенджер + соцсеть" — конкурент VK
```

### 6.1 Безопасность и шифрование

| Аспект | WorldMates | Telegram | Viber |
|--------|-----------|----------|-------|
| Облачные сообщения | AES-256-GCM | MTProto (сервер держит ключ) | MTProto-аналог |
| Секретные чаты | ✅ Signal Protocol (75%) | ✅ Полный | ✅ Полный |
| Forward Secrecy | ✅ (когда Signal включён) | Только в секр.чатах | ✅ По умолч. |
| Open Source | ❌ | ✅ (частично клиент) | ❌ |
| Bug Bounty программа | ❌ | ✅ $100K | ❌ |
| Проверка безопасности | ❌ Самостоятельно | ✅ Регуляр. аудиты | ✅ |
| Анонимность | ❌ Привязка к аккаунту соцсети | ✅ По номеру | По номеру |

**Вердикт:** Если завершить Signal Protocol — по шифрованию будете лучше Telegram (облачные чаты у них не E2EE!), на уровне Viber.

### 6.2 Производительность и масштабируемость

| Метрика | WorldMates (текущая) | Telegram | Viber |
|---------|---------------------|----------|-------|
| MAU (пользователей) | Стартап | 950M+ | 900M+ |
| Датацентры | 1 | 100+ | 10+ |
| Одновременных соединений | ~15-25K | Сотни миллионов | Десятки миллионов |
| Доставка сообщения (задержка) | ~50–200ms (local) | ~30–80ms | ~50–100ms |
| Время доставки offline | При реконнекте | Push (мгн.) | Push (мгн.) |
| Push-уведомления | Socket.IO (слабо) | FCM + APNs | FCM + APNs |
| Отказоустойчивость | 0% (1 сервер) | 99.95% | 99.9% |

**Критическая проблема:** Кастомные Socket.IO уведомления без Firebase — элегантное решение для экономии, но при выключенном экране Android убивает фоновые процессы. Пользователи будут пропускать сообщения. Нужна интеграция FCM (бесплатно) хотя бы как fallback.

**Что делает WorldMates уникальным (ваши козыри):**
1. Интеграция с соцсетью worldmates.club — нигде больше нет
2. Гибридное шифрование (E2EE + совместимость с веб) — инновация
3. Bot API с FSM — реально конкурентен с Telegram
4. Геодискавери пользователей — уникальная функция
5. 4 сервиса для облачного бэкапа — лучший выбор в индустрии
6. Livestreams в каналах — редкость среди мессенджеров
7. Видеоплеер с 6 жестами — лучший в классе

**Что мешает конкурировать прямо сейчас:**
1. Нет iOS — потеря половины рынка
2. Нет Web-клиента — люди хотят писать с ПК
3. Незавершённый E2EE — доверие пользователей
4. Нет Firebase Push — пользователи пропускают сообщения
5. 1 сервер = нет надёжности
6. Нет мониторинга = баги на проде незаметны
7. Самоудаляющиеся сообщения — базовое ожидание

**Реалистичная оценка:**
Архитектурно проект сделан **правильно и амбициозно**. Объём кода (14K строк Android + 15K строк Node.js + 235 PHP endpoints) — это работа целой команды. Если завершить E2EE, добавить iOS/Web, Firebase Push и нормальный мониторинг — это полноценный конкурент регионального уровня, который может занять нишу "Telegram + ВКонтакте в одном флаконе" для конкретных рынков.
